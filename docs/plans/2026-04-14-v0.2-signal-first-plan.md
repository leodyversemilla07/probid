# probid v0.2 Signal-First Minimal Roadmap

> For Hermes: implement in small, reversible increments. Keep backward compatibility for existing commands.

## Goal

Ship a v0.2 that feels minimal like pi.dev while improving investigation signal quality.

## Constraints

- Keep CLI simple and local-first
- Preserve existing command behavior
- Use RA 12009 + IRR baseline for legal semantics
- Maintain explainable, auditable heuristics
- Avoid paid dependencies

## Selected approach

Signal-first minimal roadmap with progressive disclosure:
- Summary-first outputs for quick triage
- Evidence-rich detail only on demand
- Reason-coded flags with confidence and caveats

---

## Scope for v0.2

### In scope
- Add `probe` command as the primary front-door workflow
- Add reason code system (R1-R8) for heuristic outputs
- Add confidence and false-positive notes in analysis output
- Add `--why` and `--json` for machine-readable and detailed evidence
- Keep all existing commands intact

### Out of scope
- Web dashboard
- New external paid APIs
- Full legal inference engine
- Breaking command interface changes

---

## Reason code catalog (v0.2)

- R1: Repeat supplier concentration
- R2: Near-ABC award pattern
- R3: Potential split contracts in short interval
- R4: Procurement mode outlier frequency
- R5: Abnormal budget utilization spread for similar category
- R6: Single-agency dependence risk (supplier)
- R7: Sparse/low-confidence data warning
- R8: Beneficial ownership disclosure gap (when data exists)

Each reason code must include:
- Trigger condition
- Evidence fields used
- Confidence (low/medium/high)
- False-positive note

---

## Data contract for `probe`

Input:
- Query keyword (required)
- `--agency` optional filter
- `--pages` (default 1)
- `--why` include evidence snippets
- `--json` emit structured output

Output sections (default text mode):
1. Summary
   - Query, records scanned, agencies touched, total known value
2. Top findings
   - Up to 5 findings sorted by severity then confidence
3. Risk map
   - Count by reason code (R1..R8)
4. Next checks
   - Suggested follow-up commands (`detail`, `agency`, `supplier`)

JSON mode should include:
- metadata
- summary
- findings[] with reason_code, confidence, evidence, caveats
- generated_at

## Risks and mitigations

Risk: Signal inflation due to noisy heuristics
- Mitigation: confidence tiers + caveat text + capped top findings

Risk: CLI complexity creep
- Mitigation: one front-door command, advanced paths remain optional

Risk: Source scraping instability
- Mitigation: cache-first fallback and explicit uncertainty labeling

Risk: Legal misinterpretation
- Mitigation: treat outputs as risk flags, not legal conclusions
