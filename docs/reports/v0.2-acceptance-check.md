   - In this shell, `probid` was not available directly; `uv run probid` was required.
   - Follow-up: document this more explicitly for source checkouts, or ensure editable install guidance makes `probid` available.

2. **`detail` cache flag mismatch**
   - Acceptance text says `detail <ref>` should work with cached and forced modes, but `detail` does not support `--cache-only`.
   - Current behavior: cached data is used automatically when available, and `--force` exists for re-fetch.
   - Follow-up: either update acceptance wording to match behavior or add a `--cache-only` flag to `detail` for consistency.

3. **Terminal encoding/rendering**

   - Follow-up: consider ASCII-safe rendering fallback or clearer placeholders for empty numeric fields.
